# Studi Kasus: Pengelolaan Sampah di Lingkungan RT/RW

## 1. Identifikasi Masalah

Berdasarkan pengamatan pada lingkungan RT/RW skala kelurahan, ditemukan beberapa masalah berulang terkait pengelolaan sampah:

1. **Jadwal pengangkutan tidak transparan** — warga baru RT/RW atau warga yang sibuk sering tidak tahu hari dan jam pasti pengangkutan sampah di lingkungannya.
2. **Tidak ada saluran pelaporan resmi** — ketika ada tumpukan sampah liar, sampah terlewat jadwal, atau pembuangan sembarangan, warga hanya mengandalkan grup WhatsApp yang informasinya cepat tenggelam.
3. **Pengurus RT/RW kesulitan memantau** — tanpa pencatatan terstruktur, pengurus tidak punya gambaran menyeluruh titik mana yang bermasalah dan mana yang sudah ditangani.
4. **Tidak ada bukti/riwayat penanganan** — sulit mengevaluasi seberapa cepat laporan warga ditindaklanjuti karena tidak tercatat.

## 2. Dampak

- Sampah menumpuk lebih lama dari seharusnya → bau, sumber penyakit, mengganggu estetika lingkungan
- Ketidakpercayaan warga terhadap kinerja pengurus RT/RW
- Potensi konflik antar warga akibat pembuangan sampah sembarangan yang tidak tertangani

## 3. Solusi yang Diusulkan

**Sigap Sampah** — aplikasi web sederhana dengan dua sisi pengguna:

- **Warga**: dapat melihat jadwal pengangkutan per RT dan mengirim laporan titik sampah bermasalah secara terstruktur (bukan lewat chat yang mudah hilang).
- **Pengurus RT/RW (Admin)**: dapat mengelola status setiap laporan (Menunggu → Diproses → Selesai) dan mengatur jadwal pengangkutan per RT, sehingga seluruh riwayat penanganan tercatat dan dapat dipantau.

## 4. Mengapa Ini Relevan sebagai Studi Kasus RPL

Kasus ini dipilih karena:
- Merupakan masalah nyata di kehidupan sehari-hari (bukan kasus abstrak)
- Cukup sederhana untuk diselesaikan dalam skala tugas kuliah, namun tetap membutuhkan tahapan RPL lengkap: analisis kebutuhan, desain (use case, ERD), implementasi bertahap, dan pengujian
- Cocok dikelola dengan tools manajemen proyek seperti GitHub Issues/Projects karena fiturnya dapat dipecah menjadi task-task kecil (form pelaporan, halaman jadwal, dashboard admin, dsb.)

## 5. Metode Pengumpulan Data (opsional, isi sesuai kondisi kelompok)

- Wawancara singkat dengan ketua RT/RW setempat
- Observasi langsung titik-titik pembuangan sampah di lingkungan
- Survei kecil ke warga terkait kepuasan terhadap jadwal pengangkutan saat ini
